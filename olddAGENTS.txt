## Projet
**"Analyse Comparative des Solutions ASR pour le Traitement Automatique des Appels d'Urgence Multilingues"**
- PFE M2 SDAD — Université A. MIRA de Béjaïa / Protection Civile (DGPC) Béjaïa
- 3 objectifs : (1) Diplôme, (2) Label Startup "Data Algérie", (3) Plateforme décisionnelle DGPC

## Architecture Pipeline (décidée, ne pas changer)
```
Audio → Whisper V3 FT (kabyle) → Transcription
                                      ↓
Audio → openSMILE (prosodie)    → stress_score
                                      ↓
Transcription → Qwen 2.5 7B QLoRA + Outlines → JSON structuré
                                      ↓
JSON → Post-processing déterministe (compute_urgency, compute_dispatch)
                                      ↓
                              EmergencyCall (output final)
```

## Séparation des responsabilités (CRITIQUE)
- **Le LLM extrait les FAITS** : `incident_type`, `location`, `victims_count`, `injury_severity`, `fire_present`, `trapped_persons`, `weapons_involved`, `hazmat_involved`
- **Le CODE calcule les DÉCISIONS** : `urgency` (compute_urgency, 16 règles), `dispatch` (compute_dispatch, déterministe)
- **Ne jamais demander au LLM de produire** : urgency, dispatch, assets, staffing — ce sont des computed fields

## Build, Test, and Development Commands
- **Setup** : `python -m venv .venv && .venv\Scripts\activate` puis `pip install -r requirements.txt`
- **Annotation app** : `cd annotation_app && streamlit run dgpc_annotation_local.py`
- **Pipeline** : depuis `ml_pipeline/`, ex: `python dgpc_pipeline.py`
- **Tests** : `pytest` avec fichiers `test_*.py`

## Coding Style
- **Python** : 4 espaces, snake_case fonctions/modules, PascalCase classes. Type hints obligatoires. Format `black`, lint `ruff`.
- **Enums** : toujours en ANGLAIS (codes machine). Labels français uniquement dans l'UI.
- **Data/assets** : ne PAS commit audio brut, modèles lourds, CSV/JSON générés. Placer nouveaux datasets dans sous-dossiers dédiés.
